# MES_STD 資料夾理解確認

文件日期：2026-05-06

## 目的

這份文件用來確認我們和使用者對 `MES_STD` 資料夾的理解是否一致，尤其是：

- `2026` 資料夾底下是否應該混放很多不同日期的資料
- 哪些資料會產出 Query2 Excel
- 哪些資料會產出 TO14C CSV
- 目前錯誤資料是程式問題、基本參數檔問題，還是資料來源輸入問題

目前樣本來源：

```text

```

底下不是每日資料夾，而是直接放來源資料夾：

```text

  PORT 5\
  PORT 11\
```

實際的日期藏在 `.D` 資料夾名稱中，例如：

所以程式目前不是用 `2026` 判斷同一天，而是用 `.D` 名稱中的 `[yyyyMMdd HHmm]` 判斷批次日期。

## Excel 產出規則

Query2 Excel 是依「批次日期」產出。

只要該日期資料成功匯入，就會產出一份：

```text
Cylinder_Qc[yyyyMMdd].xlsx
```

例如：

```text
Cylinder_Qc[20260430].xlsx
Cylinder_Qc[20260505].xlsx
```

依目前樣本，理論上最多會有 `8` 份 Query2 Excel，對應 8 個批次日期。

但如果該日期有錯誤，例如 LOT 查不到、Misc 沒有 LOT、RF 查不到，該日期就不會產正式 Query2 Excel，而會進錯誤報表。

## CSV 產出規則

TO14C CSV 不是每天一份。

CSV 是依「成功產出的 PORT_PPB row」產出。簡化理解：

- 該日期要有可用 STD AVG
- 該 PORT / LOT 至少要有 2 筆資料可算 AVG
- LOT 要存在於 `ZZ_NF_GAS_MFG_LOT`
- RF 要存在於 `ZZ_NF_GAS_QC_RF`
- DB 寫入成功

依目前資料排列推估，若 LOT/RF 都補齊且匯入成功，可能產 CSV 的資料如下：

| 批次日期 | CSV 來源 |
| --- | --- |
| 20260429 | PORT 4 / 20260428007 |
| 20260429 | PORT 5 / 20260428008 |
| 20260430 | PORT 11 / 20260428009 |
| 20260430 | PORT 12 / 20260428010 |
| 20260430 | PORT 2 / 20260428011 |
| 20260430 | PORT 3 / 20260428012 |
| 20260430 | PORT 4 / 20260428013 |
| 20260430 | PORT 5 / 20260429001 |
| 20260430 | PORT 11 / 20260429002 |
| 20260430 | PORT 12 / 20260429003 |

所以 CSV 不是 8 份，而是依成功的 PORT_PPB 筆數，這批資料推估最多約 `10` 份。

## 目前看到的異常點

### 1. 同一個 `2026` 資料夾混了很多日期

目前同一層資料包含：

```text
20260205
20260313
20260325
20260413
20260429
20260430
20260504
20260505
```

需要確認這是否符合使用者預期。

如果使用者原本以為 `2026` 底下只會有當天資料，那目前資料夾內容和理解有落差。

### 2. 部分 LOT 看起來不像原本預期的 11 碼 LOT

例如：

```text
63265
150703
406602
```

這些值是從 Quant.txt 的 `Misc` 欄位讀到的：

```text
Misc : port 13  40>  #63265
Misc : port 15  40>  #150703
```

程式規則是抓 `#` 後面的內容當 LOT，所以會拿 `63265`、`150703` 去查 `ZZ_NF_GAS_MFG_LOT`。

需要確認：

- 這些短碼是否就是合法 LOT？
- 還是使用者基本參數檔或機台輸入填錯？
- 若是短碼，DB 的 `ZZ_NF_GAS_MFG_LOT` 是否也要建立這些短碼？

### 3. 有些 Quant.txt 沒有 LOT

例如：

```text
Misc : port 1  903
Misc : port 12  045  1146>  #
```

這兩種都無法解析出 LOT。

需要使用者補正 `Misc`，例如：

```text
Misc : port 12  045  1146>  #20260505001
```

### 4. 有些 `.D` 資料夾名稱沒有 `_V006` 或 `_903`

例如：

```text
PORT 12[20260505 1732].D
STD[20260504 1047].D
```

原本程式從資料夾尾端 `_V006` 或 `_903` 取 SampleNo。

現在已調整為：如果資料夾名稱沒有 SampleNo，就改從 Quant.txt 的：

```text
Data File : 0001.D
```

解析 SampleNo。

## 需要使用者確認的問題

1. `MES_STD\2026` 是否本來就會累積整年度資料，而不是每日資料夾？
2. `63265`、`150703`、`406602` 這種短碼是否是合法 LOT？
3. 若短碼是合法 LOT，`ZZ_NF_GAS_MFG_LOT` 是否要建立對應資料？
4. `20260504`、`20260505` 這些 `Misc` 沒有 LOT 的資料，是否代表基本參數檔沒設定好？
5. 使用者期望 CSV 是「每天一份」還是「每個 PORT_PPB / sample 一份」？
6. Query2 Excel 是否可以接受「每個批次日期一份」？

## 目前程式處理方式

- 監聽 `MES_STD\2026`
- 不搬移原始 `.D`
- 用 `.D` 名稱中的 `[yyyyMMdd HHmm]` 判斷批次日期
- 成功資料輸出到獨立輸出資料夾：

```text
C:\JinZhaoYi_Output\MES_STD\2026\QC
```

- 異常資料產出錯誤報表：

```text
ImportErrors[yyyyMMddHHmmss].xlsx
```

錯誤報表會列出每一筆檔案自己的 LOT、路徑、錯誤訊息與建議處理方式，避免把同一批次的所有錯誤 LOT 混在同一列。
